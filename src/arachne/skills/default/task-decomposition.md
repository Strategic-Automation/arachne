# Task Decomposition Guidelines

When breaking down complex goals into sub-tasks:

1. **Start with the end** — Define the final deliverable first.
2. **Work backwards** — What information or artifacts are needed for that deliverable?
3. **Parallelize** — Identify independent sub-tasks that can run simultaneously.
4. **Sequence** — Order dependent tasks so outputs flow naturally as inputs to the next step.
5. **Scope** — Each node should have a single, focused responsibility. Max 1-2 outputs.

If the goal requires external data, always include a research/fetch node before analysis nodes.
